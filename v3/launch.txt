How to Make It Work – Step‑by‑Step Setup
Prerequisites
Python 3.10+

Docker & Docker Compose (for Kafka + Redis)

Root/administrator privileges (for packet sniffing)

libpcap (Linux) or Npcap (Windows) – scapy needs it

1. Clone & Install Dependencies
bash
git clone <your-repo> dns_pipeline_v3
cd dns_pipeline_v3
python -m venv venv
source venv/bin/activate   # or .\venv\Scripts\activate on Windows
pip install -r requirements.txt
2. Start Infrastructure (Kafka + Redis)
bash
docker-compose up -d
This spins up:

Kafka (with Zookeeper) on localhost:9092

Redis on localhost:6379

Wait 15 seconds for Kafka to fully start.

3. (Optional) Train or Place a Model
If you have a trained sklearn model (e.g., RandomForest on the 16 features), save it as model/model.pkl.

Otherwise the system automatically falls back to the heuristic classifier (model/heuristic.py). No model is required.

4. Start the API Gateway
bash
# From project root
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
Check health: curl http://localhost:8000/health

5. Start Workers (one or many)
In a new terminal:

bash
# Single worker
python worker/consumer.py

# Multiple workers (scale horizontally)
# Run each in its own terminal, or use docker-compose scale
python worker/consumer.py --consumer-group worker_group_1
Workers will automatically start consuming from dns_windows.


 ## I tried the above but it didnt work, instead:

You already have a worker service in docker-compose.yml. Start it with:

bash
docker compose up -d worker
If you want multiple workers:

bash
docker compose up -d --scale worker=3


5,5:
The worker is running, but the API is not. The CLI sends windows to http://localhost:8000 – that’s the API. Start it:

bash
docker compose up -d api
Verify it’s healthy:

bash
curl http://localhost:8000/health


6. Run a CLI Agent (sniffing)
On Linux (requires sudo for packet capture):

bash
sudo python cli/main.py --interface eth0
On Windows (run as Administrator):

bash
python cli/main.py --interface "Wi-Fi"
Additional options:

--api-url http://localhost:8000 (if API on different host)

--agent-id sensor_rack01 (default = hostname)

--no-sniff (for testing with PCAP replay)

7. Watch the Dashboard
The CLI will show a Rich live table:

One row per 1‑second window

Columns: window_id, timestamp, queries, unique_domains, label, confidence

Green = benign, red = malicious

Every window is classified within ~1‑2 seconds (polling every 200ms).

8. Testing Without Real Traffic
You can replay a PCAP file:

bash
sudo python cli/main.py --replay sample_dns.pcap --speed 1.0
Or generate synthetic DNS DGA traffic using scapy / dnsgrep and run the sniffer.

9. Stopping Everything
Ctrl+C on CLI agent → threads exit cleanly.

Ctrl+C on worker → commits current batch, then exits.

Ctrl+C on API → FastAPI shuts down.

docker-compose down to stop Kafka + Redis.
